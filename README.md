# Botium Toys Security Audit Case Study

This repository contains a small security audit case study I completed during the Google Cybersecurity Certificate.

The project is based on a fictional company, Botium Toys. Using the scenario provided in the course, I reviewed the company’s existing controls, looked at basic compliance-related issues, and identified some of the main risks in the environment.

This was a simulated academic project, not a real client engagement.

## Repository contents

- `botium-toys-completed-checklist.pdf` — completed controls and compliance checklist
- `README.md` — short overview of the case study

## What I reviewed

- existing security controls
- missing or weak controls
- basic compliance-related gaps
- risks affecting customer and company data

## Main takeaways

Some controls were already in place, such as a firewall, antivirus, physical locks, CCTV, and fire detection systems.

The main gaps identified in the scenario included missing least privilege, no separation of duties, no intrusion detection system, no backups or disaster recovery plan, no encryption for cardholder data, and weak password management practices.
